# Section 1: What Are Embedded Commands?

## Core Content
An embedded command is a suggestion hidden inside a larger sentence so the listener's conscious mind hears a normal statement, while the unconscious mind picks up the instruction.

## Key Points
- Hidden inside longer sentences
- Conscious mind hears a statement, unconscious receives the command
- Marked by tone shifts, pauses, slower speech, eye contact
- Bypasses resistance

## Example
"You might find that as you listen, you can begin to relax now." - The command is "relax now" hidden inside a conversational sentence.

## Comprehension Question
What is the difference between a direct command and an embedded command?

## Follow-up Guidance
If they struggle, explain: Direct commands like "Relax!" trigger resistance. Embedded commands slip past the conscious mind because they sound like normal conversation.

# Section 2: Why They Work

## Core Content
Embedded commands work because of three psychological effects: the conscious vs unconscious split, reactance theory, and cognitive load. People resist being told what to do—but they accept ideas they feel they chose themselves.

## Key Points
- Conscious mind analyzes logically, unconscious processes patterns
- Indirect suggestions reduce reactance
- When sentence is long, conscious focuses on meaning, not structure

## Example
Direct: "Relax." → Resistance!
Indirect: "Some people notice as they consider this idea, they naturally begin to trust their judgment." → Accepted!

## Comprehension Question
Why does the unconscious mind accept embedded commands more easily than direct commands?

## Follow-up Guidance
If they struggle, explain: The conscious mind is busy processing the surface meaning of the sentence, so it doesn't notice the hidden command being installed.

# Section 3: The Structure

## Core Content
The basic structure is: Frame + Permission + Hidden Action. Example: "You might discover that as you consider this idea, you can feel more confident now."

## Key Points
- Frame: "You might discover that..."
- Permission: "as you consider this idea..."
- Hidden Action: "you can feel more confident now"

## Example
"You may notice how easily this can change." - Frame + Hidden Action (no explicit permission needed)

## Comprehension Question
What are the three components of the embedded command structure?

## Follow-up Guidance
If they struggle, remind them: Frame gives context, Permission makes it safe, Hidden Action is what you want them to do.

---

# Section 4: How to Deliver

## Core Content
To deliver embedded commands effectively: hide the command in a longer sentence, emphasize subtly with voice/pause/eye contact, and keep it natural.

## Key Points
- Use slower voice on the command word
- Small pause before and after
- Eye contact on the command
- Must sound conversational, not scripted

## Example
"As you consider this → (pause) → you can trust yourself now."

## Comprehension Question
What are three ways to mark an embedded command non-verbally?

## Follow-up Guidance
If they struggle, suggest: Tone shift (slower on command), pause (brief silence), eye contact (connect when delivering).
